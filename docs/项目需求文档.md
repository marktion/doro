# Doro - Claude Code 语音交互界面

## 1. 项目概述

**目标**：为 Claude Code 提供语音交互能力

**核心流程**：
```
用户说话 → MiMo ASR识别 → Agent SDK → Claude Code输出 → MiMo TTS合成 → 语音播放
```

**项目本质**：Claude Code 的语音交互前端
- 把命令行的"文字输入"改成"语音输入"
- 把命令行的"文字输出"改成"语音播放" + "动态UI"

## 3. API接口

### 3.1 MiMo ASR（语音识别）

- **端点**：`POST https://api.xiaomimimo.com/v1/chat/completions`
- **模型**：`mimo-v2.5-asr`
- **认证**：`api-key: {MIMO_API_KEY}`
- **音频格式**：仅支持 WAV 和 MP3，必须是 Data URL 格式（`data:audio/wav;base64,...`）
- **响应**：一次性返回最终识别结果（非流式）
- **请求格式**：
  ```json
  {
    "model": "mimo-v2.5-asr",
    "messages": [{
      "role": "user",
      "content": [{
        "type": "input_audio",
        "input_audio": {
          "data": "data:audio/wav;base64,..."
        }
      }]
    }],
    "asr_options": {
      "language": "zh"
    }
  }
  ```

### 3.2 MiMo TTS（语音合成）

- **端点**：`POST https://api.xiaomimimo.com/v1/chat/completions`
- **模型**：`mimo-v2.5-tts`
- **认证**：`api-key: {MIMO_API_KEY}`
- **请求格式**：
  ```json
  {
    "model": "mimo-v2.5-tts",
    "messages": [
      { "role": "user", "content": "语音风格描述" },
      { "role": "assistant", "content": "要合成的文本" }
    ],
    "audio": { "format": "wav", "voice": "Chloe" }
  }
  ```
- **响应**：Base64编码的音频数据

### 3.3 WebSocket协议

**传输模式**：混合文本（JSON）+ 二进制（ArrayBuffer）

**客户端 → 服务器**：

| 消息类型 | 说明 | 数据 |
|---------|------|------|
| `query` | 发送查询 | `{ prompt, sessionId? }` |
| `query:interrupt` | 中断查询 | `{}` |
| `session:list` | 列出会话 | `{}` |
| `session:messages` | 获取会话消息 | `{ sessionId }` |
| `voice:audio` | 发送语音 | `{ data: base64, mimeType }` |
| `tts:play` | 请求TTS | `{ text }` |
| `ping` | 心跳 | `{ timestamp }` |

**服务器 → 客户端**：


| 消息类型 | 说明 | 数据格式 |
|---------|------|----------|
| `agent:message` | Agent消息 | JSON文本 |
| `asr:result` | ASR结果 | JSON文本 |
| `tts:audio` | TTS音频元数据 | JSON文本 `{ sentence, size, silenceGapMs }` |
| `tts:audio` | TTS音频数据 | **二进制 ArrayBuffer** |
| `tts:done` | TTS完成 | JSON文本 |
| `error` | 错误 | JSON文本 |
| `pong` | 心跳响应 | JSON文本 |

## 4. 功能模块

### 4.1 语音输入模块

**文件**：`client/src/composables/useVoiceRecorder.ts`

- 使用 `AudioContext` + `ScriptProcessorNode` 录音
- 输出 WAV 格式（16kHz，单声道，16bit PCM）
- 按住说话 / 松开发送
- 支持鼠标和触摸事件

**音频参数**：
| 参数 | 值 |
|------|-----|
| 采样率 | 16000 Hz |
| 声道 | 单声道 |
| 位深 | 16 bit PCM |
| 格式 | WAV |

### 4.2 语音输出模块

**文件**：`client/src/composables/useVoicePlayer.ts`

- 使用 Web Audio API 播放
- 全局 AudioContext 实例（页面生命周期内复用）
- 支持浏览器自动播放策略解锁
- 播放队列：支持段间静默间隙（`silenceGapMs` 参数）
- 淡入淡出：20ms fade消除爆破音
- 固定长度分块：80-120字，优先标点切分，兜底硬切

**浏览器自动播放策略处理**：
1. 创建全局 AudioContext 实例
2. 在用户首次交互（按住说话按钮）时调用 `unlockAudioContext()`
3. 解锁后整个页面生命周期内可自动播放 TTS 音频

### 4.3 Agent SDK集成

**文件**：`server/src/sdk/agent.ts`

- `runAgent(prompt, options)` - 异步生成器，yield SDKMessage
- `listSessions(dir)` - 列出会话
- `getSessionMessages(sessionId)` - 获取会话消息
- `deleteSession(sessionId)` - 删除会话

### 4.4 WebSocket消息路由

**文件**：`server/src/ws/handler.ts`

- `query` → Agent SDK查询（支持中断）
- `voice:audio` → ASR识别（base64 → Data URL）
- `tts:play` → TTS合成（句子级分块，文本信令 + 二进制音频）
- `session:*` → 会话管理

## 8. 关键设计决策

1. **Agent SDK而非PTY**：结构化消息、完整会话管理、生成式UI支持、Windows兼容
2. **单端口**：Express和WebSocket共享同一端口(3000)
3. **动态目录**：使用`process.cwd()`获取工作目录，传递给SDK的dir参数
4. **WAV录音格式**：使用AudioContext录制16kHz单声道WAV，ASR API仅支持WAV/MP3
5. **句子级TTS**：按标点分句，降低延迟，支持流式播放
6. **混合传输模式**：WebSocket文本(JSON)传信令，二进制(ArrayBuffer)传音频，避免Base64膨胀33%
7. **浏览器自动播放策略**：利用用户首次交互解锁AudioContext，之后可自动播放TTS
8. **TTS分块与播放**：80-120字固定长度分块，段间500ms静默，20ms淡入淡出，确保语速稳定、播放流畅
9. **会话状态持久化**：用户消息、当前会话ID、选中会话保存到localStorage，刷新页面不丢失历史

## 9. 已知问题与解决方案

### 9.1 ASR识别失败：Bare base64 is not supported

**问题**：MiMo ASR API要求Data URL格式，不支持纯Base64字符串

**解决**：前端录制WAV后，后端构建完整Data URL：`data:audio/wav;base64,...`

### 9.2 TTS播放失败：DOMException: Suspended value failed

**问题**：浏览器自动播放策略禁止无用户交互时播放音频

**解决**：
1. 创建全局AudioContext实例
2. 在用户首次交互（按住说话）时调用 `unlockAudioContext()`
3. 解锁后整个页面生命周期内可自动播放

### 9.3 WebSocket二进制数据解析失败

**问题**：WebSocket同时传输JSON文本和二进制音频，未正确区分类型

**解决**：
1. 设置 `ws.binaryType = 'arraybuffer'`
2. `onmessage` 中先判断数据类型（ArrayBuffer/Blob/字符串）
3. 二进制数据直接走音频处理，不执行JSON.parse()
